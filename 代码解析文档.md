## 三、模块详解

### 3.1 robot.py — 核心协调器（最重要）

**职责：** 管理整个对话生命周期，是整个系统的"大脑"。

**关键设计：**

```python
class Robot:
    def __init__(self, config_file):
        # 通过工厂模式创建各模块实例
        self.recorder = recorder.create_instance(...)
        self.asr      = asr.create_instance(...)
        self.llm      = llm.create_instance(...)
        self.tts      = tts.create_instance(...)
        self.vad      = vad.create_instance(...)
        self.player    = player.create_instance(...)
```

**工厂模式解耦：**
```python
def create_instance(class_name, *args, **kwargs):
    cls = globals().get(class_name)  # 根据字符串名动态获取类
    return cls(*args, **kwargs)
```
配置文件里写 `ASR: FunASR`，代码就实例化 `FunASR`，非常易于换模块。

**主循环 `run()`：**
```python
def run(self):
    self.start_recording_and_vad()   # 启动录音 + VAD 线程
    while not self.stop_event.is_set():
        self._duplex()  # 双工处理（核心）
```

**双工处理 `_duplex()` — 核心逻辑：**
```python
def _duplex(self):
    data = self.vad_queue.get()          # 取出 VAD 判断结果
    vad_status = data.get("vad_statue")  # start / end / None

    if "start" in vad_status:
        self.speech.append(data)         # 说话开始，收集音频
    elif "end" in vad_status:
        voice_data = [d["voice"] for d in self.speech]
        text, tmpfile = self.asr.recognizer(voice_data)  # ASR 识别
        self.executor.submit(self.chat, text)  # 提交对话任务
        self.speech = []
```

**打断机制 `interrupt`：**
```python
if self.player.get_playing_status():   # TTS 正在播放
    if self.INTERRUPT:                  # 启用了打断
        self.interrupt_playback()       # 停止播放
        self.vad_start = True            # 开始新的录音
```

**TTS 优先级队列：**
```python
# 主线程把 TTS 任务放进队列
future = self.executor.submit(self.speak_and_play, text)
self.tts_queue.put(future)

# 独立线程顺序播放（保证不乱序）
def _tts_priority(self):
    while not self.stop_event.is_set():
        future = self.tts_queue.get()    # 阻塞等待
        tts_file = future.result()       # 拿 TTS 结果
        self.player.play(tts_file)       # 播放音频
```

### 3.2 recorder.py — 音频录制

**实现：** `RecorderPyAudio` 类，使用 PyAudio 从麦克风实时采集音频流。

```python
class RecorderPyAudio(AbstractRecorder):
    def start_recording(self, audio_queue):
        self.stream = self.py_audio.open(
            format=pyaudio.paInt16,
            channels=1,          # 单声道
            rate=16000,          # 16kHz（Whisper/VAD 标准采样率）
            input=True,
            frames_per_buffer=512  # 每次读取 512 帧（约 32ms）
        )
        while self.running:
            data = self.stream.read(self.chunk)
            audio_queue.put(data)   # 实时放入队列
```

**关键参数：**
- `rate = 16000` — 16kHz 采样率，是 ASR 和 VAD 的标准要求
- `chunk = 512` — 每块 512 帧 ≈ 32ms延迟
- `format = paInt16` — 16位整数，标准音频格式

### 3.3 vad.py — 语音活动检测

**实现：** `SileroVAD`，使用 Silero-VAD 模型判断语音的起始和结束。

```python
class SileroVAD(VAD):
    def is_vad(self, data):
        audio_int16 = np.frombuffer(data, dtype=np.int16)
        audio_float32 = self.int2float(audio_int16)        # int16 → float32
        vad_output = self.vad_iterator(torch.from_numpy(audio_float32))
        return vad_output  # {"start": T} 或 {"end": T} 或 None
```

**返回值的含义：**
- `{"start": T}` — 检测到语音开始
- `{"end": T}` — 检测到语音结束（停顿超过 `min_silence_duration_ms`）
- `None` — 仍在说话中

**配置说明：**
```yaml
VAD:
  SileroVAD:
    sampling_rate: 16000
    threshold: 0.5         # 灵敏度，越低越灵敏
    min_silence_duration_ms: 200  # 停顿多久算说完
```

### 3.4 asr.py — 语音识别

**实现：** `FunASR` 类，内部使用 SenseVoiceSmall 模型。

```python
class FunASR(ASR):
    def __init__(self, config):
        self.model = AutoModel(
            model="FunAudioLLM/SenseVoiceSmall",  # 或本地路径
            vad_kwargs={"max_single_segment_time": 30000},
            hub="hf"
        )

    def recognizer(self, stream_in_audio):
        # 1. 把原始音频存成 WAV 文件
        tmpfile = save_audio_to_wav(stream_in_audio)
        # 2. FunASR 识别
        res = self.model.generate(input=tmpfile, language="auto", use_itn=True)
        text = rich_transcription_postprocess(res[0]["text"])
        return text, tmpfile
```

**为什么存文件？** FunASR 的 `generate()` 接受文件路径而非实时流，所以先把实时音频段缓存成 WAV 再识别。这是它与 Whisper 流式处理的一个关键区别。

**`use_itn=True`** = 使用 Inverse Text Normalization，把"一百二十三"还原成"123"等数字形式。

### 3.5 llm.py — 大语言模型

**实现：** `OpenAILLM` 类，封装 OpenAI 兼容 API。

```python
class OpenAILLM(LLM):
    def response(self, dialogue):
        responses = self.client.chat.completions.create(
            model=self.model_name,    # deepseek-chat / gpt-4o-mini 等
            messages=dialogue,        # 对话历史
            stream=True                # 流式返回
        )
        for chunk in responses:
            yield chunk.choices[0].delta.content  # 逐 token yield
```

**流式处理的好处：** 第一个 token 出来就开始 TTS，不等完整回复，延迟降低。

**带工具调用的版本 `response_call()`：**
```python
def response_call(self, dialogue, functions_call):
    responses = self.client.chat.completions.create(
        model=self.model_name,
        messages=dialogue,
        stream=True,
        tools=functions_call     # 传入插件函数定义
    )
    for chunk in responses:
        yield chunk.choices[0].delta.content, \
              chunk.choices[0].delta.tool_calls  # content + 工具调用
```

### 3.6 tts.py — 语音合成

提供了多个 TTS 实现，通过配置切换：

| 类 | 特点 |
|----|------|
| `EdgeTTS` | 微软 TTS，延迟低，音质好，**推荐** |
| `CHATTTS` | 开源本地 TTS，效果好但需 GPU |
| `GTTS` | Google TTS，需联网 |
| `MacTTS` | macOS 系统语音 |

**EdgeTTS 实现（推荐）：**
```python
class EdgeTTS(AbstractTTS):
    async def text_to_speak(self, text, output_file):
        communicate = edge_tts.Communicate(text, voice=self.voice)
        await communicate.save(output_file)

    def to_tts(self, text):
        tmpfile = self._generate_filename(".wav")
        asyncio.run(self.text_to_speak(text, tmpfile))
        return tmpfile
```

### 3.7 dialogue.py — 对话历史管理

```python
class Dialogue:
    def put(self, message: Message):
        # 添加消息到历史
        self.messages.append(message)

    def get_llm_dialogue(self):
        # 返回给 LLM 的对话格式
        return [{"role": m.role, "content": m.content} for m in self.messages]

    def dump_dialogue(self):
        # 持久化到磁盘
        with open(self.history_file, "w") as f:
            json.dump([m.__dict__ for m in self.messages], f, ensure_ascii=False)
```

## 四、工厂模式 — 如何热插拔模块

整个系统通过配置文件决定使用哪个模块实现，无需改代码：

```python
# config.yaml 中写：
selected_module:
  ASR: FunASR      # 换成 WhisperASR 也行
  VAD: SileroVAD   # 换成其他 VAD 也行
  LLM: OpenAILLM   # 换成 LocalLLM 也行
  TTS: EdgeTTS     # 换成 CHATTTS 也行
```

```python
# 各模块的 create_instance() 自动根据字符串名实例化
asr = asr.create_instance("FunASR", config)
tts = tts.create_instance("EdgeTTS", config)
llm = llm.create_instance("OpenAILLM", config)
```

**二次开发时可以新增自己的模块，只需：**
1. 在对应文件中定义类继承 ABC
2. 在 `create_instance()` 中注册
3. 在 `config.yaml` 中选用

## 五、对话管理（chat 方法解析）

```python
def chat(self, query):
    self.dialogue.put(Message(role="user", content=query))  # 加入历史
    self.chat_lock = True   # 加锁防止打断

    # 流式 LLM 调用
    for content in self.llm.response(self.dialogue.get_llm_dialogue()):
        response_message.append(content)
        if is_segment(response_message):   # 凑够一个完整句
            segment_text = "".join(response_message[start:])
            future = self.executor.submit(self.speak_and_play, segment_text)
            self.tts_queue.put(future)      # 提交 TTS 播放
            start = len(response_message)

    self.chat_lock = False   # 解锁
    self.dialogue.dump_dialogue()  # 持久化
```

**`is_segment()` 是什么：** 检查当前累积的文字是否构成一个语义完整的句子（句号/问号/感叹号等），只有完整句才触发 TTS，保证语音连贯。

## 六、工具调用（Function Calling）流程

```
用户说 "杭州天气怎么样？"
        ↓
LLM 返回 tool_calls: {"function_name": "get_weather", "args": {"city": "杭州"}}
        ↓
chat_tool() 识别到工具调用
        ↓
TaskManager.tool_call("get_weather", {"city": "杭州"})
        ↓
get_weather() 执行 → 返回天气数据
        ↓
把结果加回 dialogue，继续调用 LLM 生成最终回复
        ↓
TTS 播放回复
```
