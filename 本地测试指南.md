> 源码已 fork 至：https://github.com/yjin1989-code/ASR-TTS

> 这是一个功能完整的语音对话机器人，支持 ASR + LLM + TTS 全流程

---

## 一、项目概述

**百聆** 是一个类 GPT-4o 的实时语音对话助手，核心功能：

- 🎙️ **ASR**：FunASR（SenseVoice）— 语音转文字
- 🧠 **VAD**：Silero-VAD — 语音活动检测，判断说话开始/结束
- 💬 **LLM**：DeepSeek / OpenAI 兼容 API — 对话生成
- 🔊 **TTS**：Edge-TTS / ChatTTS / gTTS — 文字转语音
- 🛑 **打断支持**：说话时可打断正在播放的 TTS 语音
- 🧠 **记忆功能**：持续记忆对话历史
- 🔧 **工具调用**：天气查询、网络搜索、定时提醒等插件
- 📋 **任务管理**：创建和跟踪任务

## 三、环境准备

### 3.1 系统要求

| 项目 | 要求 |
|------|------|
| Python | 3.8+ |
| 操作系统 | Windows / macOS / Linux |
| GPU | 可选（无 GPU 可用 CPU 运行）|
| 麦克风 | 必须 |

### 3.2 创建虚拟环境

```bash
# 创建并激活虚拟环境
conda create -n bailing python=3.10
conda activate bailing

# 或使用 venv
python -m venv bailing-env
# Windows:
bailing-env\Scripts\activate
```

### 3.3 安装依赖

```bash
pip install -r requirements.txt
```

主要依赖包括：
- `funasr` — ASR 语音识别
- `silero-vad` — 语音活动检测
- `openai` — LLM 接口
- `edge-tts` — 微软 TTS
- `ChatTTS` — 开源 TTS
- `pygame` — 音频播放
- `pyyaml` — 配置文件解析

### 3.4 配置国内镜像（推荐）

如果 HuggingFace 下载慢，在 `pip install` 前设置：
```bash
# 设置 HuggingFace 镜像
export HF_ENDPOINT=https://hf-mirror.com

# 或者在 Python 代码中
import os
os.environ["HF_ENDPOINT"] = "https://hf-mirror.com"
```

## 四、配置

编辑 `config/config.yaml`：

```yaml
# 核心配置
WakeWord: 相一          # 唤醒词（目前未启用）
interrupt: false        # 是否启用打断功能
StartTaskMode: true     # 是否启用工具调用

selected_module:         # 选择各模块的具体实现
  ASR: FunASR           # 语音识别
  VAD: SileroVAD        # 语音活动检测
  LLM: OpenAILLM        # 大语言模型
  TTS: EdgeTTS           # 语音合成
  Player: PygameSoundPlayer  # 音频播放

ASR:
  FunASR:
    model_dir: FunAudioLLM/SenseVoiceSmall   # ASR 模型
    output_file: tmp/                          # 识别音频存放目录

VAD:
  SileroVAD:
    sampling_rate: 16000
    threshold: 0.5
    min_silence_duration_ms: 200   # 说话停顿超过200ms视为结束

LLM:
  OpenAILLM:
    model_name: deepseek-chat      # 或 gpt-4o-mini / qwen-plus 等
    url: https://api.deepseek.com  # API 地址
    api_key: 你的API密钥

TTS:
  EdgeTTS:
    voice: zh-CN-XiaoxiaoNeural    # 中文音色
    output_file: tmp/
```

### 4.1 LLM API 配置

**DeepSeek（推荐，便宜）：**
```yaml
LLM:
  OpenAILLM:
    model_name: deepseek-chat
    url: https://api.deepseek.com
    api_key: sk-your-key
```

**OpenAI 官方：**
```yaml
LLM:
  OpenAILLM:
    model_name: gpt-4o-mini
    url: https://api.openai.com/v1
    api_key: sk-your-key
```

**硅基流动（低价中转）：**
```yaml
LLM:
  OpenAILLM:
    model_name: Qwen2.5-7B-Instruct
    url: https://api.siliconflow.cn/v1
    api_key: your-key
```

## 五、运行

### 5.1 方式一：直接运行（推荐测试用）

```bash
# 进入项目根目录
cd ASR-TTS

# 运行（使用默认配置）
python main.py

# 或指定配置文件
python main.py config/config.yaml
```

运行后：
1. 控制台会打印 `Started recording.`
2. 对着麦克风说话（建议 1-2 秒后开始）
3. 系统识别语音 → 发给 LLM → TTS 播放回复
4. 按 `Ctrl+C` 退出

### 5.2 方式二：Web 服务模式

```bash
cd server
python server.py   # 启动 Web 服务
```
然后浏览器访问 http://127.0.0.1:5000

## 六、模块测试（分步验证）

建议按顺序测试每个模块是否正常工作：

### 6.1 测试 TTS（先测这个，最简单）

```python
import edge_tts
import asyncio

async def test():
    audio_file = "test.mp3"
    await edge_tts.Communicate("你好，这是百聆的语音合成测试。", 
                                "zh-CN-XiaoxiaoNeural").save(audio_file)
    print(f"TTS 生成成功: {audio_file}")

asyncio.run(test())
```

### 6.2 测试 ASR（FunASR）

```python
from funasr import AutoModel
from funasr.utils.postprocess_utils import rich_transcription_postprocess

model = AutoModel(
    model="FunAudioLLM/SenseVoiceSmall",
    disable_update=True,
    hub="hf"
)

# 识别一个音频文件
result = model.generate(
    input="你的音频文件.wav",
    language="auto",
    use_itn=True
)
text = rich_transcription_postprocess(result[0]["text"])
print(f"识别结果: {text}")
```

### 6.3 测试 VAD（Silero-VAD）

```python
from silero_vad import load_silero_vad, VADIterator
import numpy as np

vad_model = load_silero_vad()
vad_iterator = VADIterator(vad_model, threshold=0.5, sampling_rate=16000)

# int16 音频数据
audio_int16 = np.frombuffer(your_audio_bytes, dtype=np.int16)
audio_float32 = audio_int16.astype(np.float32) / 32768.0

vad_result = vad_iterator(torch.from_numpy(audio_float32))
print(f"VAD 结果: {vad_result}")
```

### 6.4 测试 LLM（快速验证 API 连通性）

```python
from openai import OpenAI

client = OpenAI(
    api_key="你的API密钥",
    base_url="API地址"
)

response = client.chat.completions.create(
    model="deepseek-chat",
    messages=[{"role": "user", "content": "你好，简短回复"}],
    stream=False

print(response.choices[0].message.content)
```

## 八、常见问题排查

### Q1: 报 `funasr` 模块找不到
```bash
pip install funasr
```

### Q2: 报 `silero-vad` 模型下载失败
```bash
# 设置镜像后重试
export HF_ENDPOINT=https://hf-mirror.com
python -c "from silero_vad import load_silero_vad; load_silero_vad()"
```

### Q3: LLM API 调用失败（429 / 401）
- 检查 `api_key` 是否正确
- 检查 `url` 是否拼写正确
- DeepSeek 有免费额度，测试足够用

### Q4: 麦克风没有声音输入
- Windows：确认麦克风权限已开启
- Linux：`sudo apt install portaudio19-dev`
- 确认系统默认录音设备正常

### Q5: TTS 播放没有声音
- 检查系统默认播放设备
- 确认 pygame 已正确安装
- 尝试换用 `MacTTS` 或 `GTTS`

### Q6: 说话结束后识别延迟高
- 调小 `VAD.min_silence_duration_ms`（如从 200 改到 100）
- 确认没有在用 CPU 模式下跑 large 模型

### Q7: 不理解用户说话的意思（LLM 回答跑偏）
- 在 `sys_prompt` 中（`robot.py` 开头）调整角色设定
- 当前设定是"开朗活泼的助手，禁止表情符号"
- 可以改成更符合你场景的 prompt

## 九、性能参考

| 模块 | 模型 | GPU 显存 | CPU 可行 |
|------|------|---------|---------|
| ASR | SenseVoiceSmall | ~1GB | ✅ |
| VAD | Silero-VAD | ~100MB | ✅ |
| LLM | DeepSeek-chat | API 调用 | ✅ |
| TTS | Edge-TTS | 0（联网）| ✅ |

> 全流程可在 **无 GPU** 的机器上运行，延迟主要来自 LLM API 响应速度。
